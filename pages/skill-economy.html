经济学视角

从供需、交易、成本、激励的角度看 Agent 时代的微技能市场

经济学 市场 Incentives

I.Skill 是什么

Skill(技能)在 Claude 的体系里是一段带 schema 的提示词 + 工具声明 + 调用入口, 在 GPT 的体系里是一份 instructions.md + 一组 tools, 在 MCP 里是一段对外暴露的服务描述。 但从经济学的视角看,Skill 是一种商品 —— 它有作者、有买家、有价格、有被淘汰的宿命。

II.供给侧

Skill 的供给曲线有一个奇怪的特点:接近零边际成本。 写一份 Skill,前几个小时沉没成本高,但复用一万次,作者每多卖一份的成本几乎为零。 这是一个教科书级的"自然垄断"候选 —— 写得好又免费的 Skill 会迅速吞掉市场。

反过来,真正稀缺的供给是作者的时间:写 Skill 的"修、配、补"成本, 还有信用的积累(这份 Skill 在我的工作流里到底能不能用)。 这两件事都不是无限可复制的。

III.需求侧

需求侧是分层裂开的:

  • to-C 尝鲜层:随便试一下,粘性极低;
  • to-P 提效层:把 Skill 嵌进工作流,愿意为省时间付钱;
  • to-B 系统层:Skill 不再是个人玩具,而是组织内部的"角色卡片",被采购、被治理。

这三层完全不同的人在想,他们的"价格弹性"差异巨大。to-C 对价格极度敏感, to-P 在意 ROI,to-B 在意合规。

IV.激励机制的反转

传统开源靠"声望"驱动贡献;Skill 时代,声望本身的捕获机制被改写了。 一份 Skill 装进 5 万个工作流,作者可能完全不知道 —— 没有"download 数"、没有 star、没有引用。

没有可见的回报,谁还愿意把最好的版本公开出来? 这是 Skill 经济的根本性未解问题。

V.平台税与抽成

每一次 Skill 调用,平台(模型厂商、Agent 平台、调用中介)都有权抽成。 这像极了 App Store 之于 iOS。但 Agent 平台的"发现机制"比应用商店模糊得多, 平台的权力比 Apple 当年还要集中(因为没有替代品)。

我的预测:未来三年,会出现一次"平台税"的舆论战。 一边是平台主张"Skill 离不开我们的模型与分发", 一边是创作者主张"价值在我们这里"。 这场战争的结果,会定义 Agent 时代的财富分配。

VI.一个判断

我赌:Skill 不会像 App 一样爆炸成"千亿市场"。 它更可能像Linux 包管理—— 重要但不性感,被少数人维护,被多数人白嫖。 真正赚钱的,不是 Skill 本身,而是 Skill 嵌入的工作流。
